# Past PR Source

Wraps the existing PR dump and synthesizes file-to-PR graph edges, PR search
chunks, review-risk hints, and file rollups.

The actual PR resources are co-located here:

- `data/current` - fetched PR metadata, comments, reviews, diffs, per-PR raw
  slices, and aggregate analysis.
- `data/prs` - processed per-PR postmortems, search indexes, known fixes, and
  run summaries.

This source slice is the bridge between PR evidence and the file graph.

Operational notes:

- `bun run pr:refresh:all` fetches the full PR corpus with 32 fetch workers and
  32 PR-review workers.
- `data/current/prs.json` is the discovered PR set for the current corpus.
- `data/current/analysis/*.jsonl` is regenerated after fetch and before graph
  rebuild.
- `data/prs/index.jsonl` is the postmortem library consumed by graph ingestion.
- `knowledge/sources/past_prs/indexes/*.jsonl` is generated by
  `kg-rebuild-graph` as the source-local graph/search index output.
- Live PR-review Pi sessions are stored under
  `decomp-orchestrator/.pi-sessions/pr-review/` for local visibility and are
  ignored by git.
